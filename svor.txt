1. Hvað gera eftirfarandi Linux skipanir?
	

	
cd
	: cd skipunnin er notuð til að breyta núverandi möppu sem er verið að vinna í
cd ~
	: fer með þig á home directory. t.d. cd~ = cd /home/username 
ls
	: skipun til að telja allar skránar
pwd
	: til að prenta vinnumöppu (möppuni sem þú ert í)
mkdir   : Stytting á "make directory", mkdir er notað til að búa til möppur í file system




2. Hvað gera eftirfarandi git skipanir:

	

git clone
	     : Klón geymslu í nýja skrá
git log
	git status
   : Sýnir commit log
git diff
	     : Sýnir skrá mismun
git checkout         : Skifta branches eða endurheimta tree files




3. Hver er munurinn á eftirfarandi git skipunum:


a) git diff 
	             : Sýnir mun á working directory og staging area
b) git diff --staged 
	     : Sýnir mun á staging area og commit1(eitthvað sem var commitað)
c) git diff commit1 commit2  : sýnir mun á commit1 og commit2(eitthvað 2 sem voru commituð)




4. Hvað er version control, hverjir eru helstu kostir við að nota GIT?

Hluti af hugbúnað af stillinga stjórnun, version control,
er stjórnun breytinga á skjölum, tölvuforritum, stór vefsíðum og öðrum söfnum upplýsinga. 
Breytingar eru yfirleitt greind með númeri eða stafakóða, 
kallað "revision number", "revision level", eða einfaldlega "revision".




5. Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu?

Eitt commit fyrir rökrétta breytingu,
þannig að gera commit fyrir eitthvað sem er bara að breyta einum hlut svo það er léttar að skylja hvað var breyt.


6. Hvernig eru Working directory, Staging area og Repository frábrugðin í GIT,
   til hvers eru þau?

Working directory: In computing, the working directory of a process is a directory of a hierarchical file system,
                   if any, dynamically associated with each process.

Staging area: The staging area is a file, generally contained in your Git directory, 
              that stores information about what will go into your next commit. 
              It’s sometimes referred to as the “index”, but it’s also common to refer to it as the staging area.

Repository: The purpose of Git is to manage a project, or a set of files, as they change over time.
            Git stores this information in a data structure called a repository. 
            A git repository contains, among other things, the following: A set of commit objects. 
            A set of references to commit objects, called heads.
